图灵机

  • 一个抽象的机器
  • 一个无限长纸带(tape)分成了一个个的小方格(cell)
  • 一个读写头(head)在纸带上移来移去
  • 读写头有一组内部状态(state)，还有一些固定的程序(action table)。在每个
    时刻，机器头都要从当前纸带上读入一个方格信息（symbol），然后结合自己的内
    部状态查找程序表，根据程序输出信息到纸带方格上，并转换自己的内部状态，然
    后进行移动。

五元组表示

$<q,b,a,m,q’>$表示图灵机的五元组

变元 含义
q 当前状态
b 当前方格原符号
a 修改后符号
m 机器头移动方向
q’ 下一状态

程序表 $f(x)=x+1$

当前状态 方格原符号 修改后符号 移动方向 下一状态
$q_1$ 0 1 L $q_2$
$q_1$ 1 0 L $q_3$
$q_1$ b b N $q_4$
$q_2$ 0 0 L $q_2$
$q_2$ 1 1 L $q_2$
$q_2$ b b N $q_4$
$q_3$ 0 1 L $q_2$
$q_3$ 1 0 L $q_3$
$q_3$ b b N $q_4$

其中

  • $q1$是起始状态；
  • $q2$表示没有进位；
  • $q3$表示有进位；
  • $q4$表示读到了空白位置，停机；
  • $b$就是blank，空白。

可计算性

如果找到一组确定的规则，按照这组规则能在有限步内求出结果，就是可计算的。图灵机存在停机问题。

图灵机不是通用的。通用图灵机是将action table和输入都写在纸带上。

小结

  1. 通过“逐层转换”，C语言程序通过电子的流动得以实现
  2. 在解决实际问题时，不必陷入“逐层转换”的细节之中：当设计一个复杂的计算机应用程序时，如设计一个文字处理软件，不需要考虑以上9个抽象层次的每一处细节，而应该集中考虑问题的本质，更多地关注从问题到算法，从算法到程序的转换
  3. 如果文字处理软件在工作时出现问题，此时，就需要了解这些细节

Data Representation in Computer System

Integer

  1. 首先将N的绝对值通过“除2取余”的方法，得到其绝对值的二进制表
    示；
  2. 如果原来的十进制数为正，则在二进制数前加0，得到k位结果；
  3. 如果原来的十进制数为负，在二进制数前加0 ，得到k位，然后再计算
    出这个补码的负数（即“取反加1”），得到结果。

Float IEEE-754标准

  • 符号位(S)
    • 0表示正数，1表示负数
  • 指数exponent 8位
  • 分数fraction 23位

1 <= exponent <= 254

$N = (-1)^s \times 1.fraction \times 2^{exponent - 127}$

  • 指数表示从-126到127
    • 为了表示负数，被表示的指数是该无符号整数减去127后得到的数值
  • 分数
    • 规格化，实际表示的是24位的精度
    • v包括23位分数和二进制小数点左端的没有明确表示出来的一位1

exponent = 00000000

  • 指数是-126，得到的有效数字以0开头
  • 后面是23位二进制分数部分
    • 分数部分全是0时，表示数值0

exponent = 11111111

  • 符号位为0，分数域全为0，表示正无穷
  • 符号位为1，分数域全为0，表示负无穷
  • 分数域不全为0，表示NaN(非数)

加法运算

  1. 对指数操作：首先使二数的指数相等(小数右移，避免舍入误差)
  2. 分数运算：经指数相等操作后，即可直接对分数做加法运算
  3. 结果规格化：对运算结果进行规格化处理
  4. 舍入操作：对丢失的位进行舍入处理
  5. 判断溢出：判断指数是否溢出

存储器

存储器是能够存储信息的二维阵列。每一行存储单元可以用唯一的标识符进行识别，地址包含一定大小的内容：指令、数据。

表示方式：$2^32 \times 8$—一共有$2^32$个存储单元，每个单元8bit.也就是有$2^32$个不同存储单元的地址空间和8位的寻址能力。可以成为4GB的存储器。
