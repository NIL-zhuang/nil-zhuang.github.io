JAVA的基本程序设计结构

数据类型

浮点类型

所有NaN的值都被认为是不同的。x==Double.NaN的结果永远是false，但是可以用Double.isNaN()方法来判断是否是非数。

char类型

Unicode转义序列会在解析代码之前得到处理，因此容易产生诡异的bug。

  1. "\u0022+\u0022"，\u0022会在解析之前变成"，也就是说得到""+""，也就是一个空串+一个空串，最后得到空串。
  2. 注释中 // \u00A0 is a new line会产生语法错误，因为\u00A0是换行符。
  3. // look inside C:/users会报错，因为/u后面跟着的不是4个十六进制数。所以路径里一定要用转义符。

    变量

用final来命名常量，常量命名方式是全大写+下划线(按照阿里命名规范)。其实const是JAVA保留字，但是目前没有使用，只用final。

运算符

strictfp

对于浮点数的运算，由于浮点寄存器的长度不同，80位的计算结果可能和64位的不一样。默认情况下采用拓展精度，但是用 strictfp关键字标记的方法必须用严格的浮点计算来生成可移植的结果。

位运算

在位运算中，&和|也可以得到Boolen值，类似于&&和||。但是他们没有短路运算的特性。

  • >>是算术移位，高位补符号位
  • >>>是逻辑移位，高位补0

字符串

JAVA字符串就是Unicode字符序列。JAVA没有内置的字符串类型，而是在标准JAVA类库中提供了一个预定义类String。

如果要多个字符串放在一起用定界符分割，用String.join("/","A","B","C")，结果是A/B/C。

字符串拼接

JAVA的字符串是不可变字符串，因此编译器可以让字符串共享。各种字符串存放在公共的存储池中，字符串变量指向存储池中对应的位置。如果复制字符串变量，原始字符串和复制的字符串共享相同的字符。共享带来的高效率远远胜过提取、拼接字符串带来的低效率。

检测字符串相等

不能用==来检测字符串是否相等。这个运算符只能确定两个字符串是否放在同一个位置上。但是如果一个字符串的多份拷贝在不同的位置上就有可能翻车。

java的compareTo方法类似于C语言中的strcmp方法(按照字典序逐个比较元素)。也可以用那个。

泛型方法

  • 所有泛型方法声明都有一个类型参数声明部分（由尖括号分隔），该类型参数声明部分在方法返回类型之前（在下面例子中的<E>）。
  • 每一个类型参数声明部分包含一个或多个类型参数，参数间用逗号隔开。一个泛型参数，也被称为一个类型变量，是用于指定一个泛型类型名称的标识符。
  • 类型参数能被用来声明返回值类型，并且能作为泛型方法得到的实际参数类型的占位符。
  • 泛型方法体的声明和其他方法一样。注意类型参数只能代表引用型类型，不能是原始类型（像int,double,char的等）。

有界的类型参数

如果想要限制被允许传递到一个类型参数的类型种类范围，就要用到有界类型参数。

要声明有界类型参数，首先列出类型参数的名称，后跟extends关键字，最后紧跟上界。
